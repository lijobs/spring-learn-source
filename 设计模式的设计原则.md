# 设计模式的设计原则

* 找出应用中可能需要变化之处，把它们独立出来，不要和那些不需要变化的代码混在一起。

* 针对接口编程，而不是针对实现编程。

* 观察者模式

  ```tex
  定义了对象之间的一对多依赖，这样一来，当一个对象改变状态时，它的所有依赖者都会收到通知并自动更新。
  
  //解释 
  主题和观察者定义了一对多的关系。观察者依赖于此主题，只要主题状态一有变化，观察者就会被通知。根据通知的风格，观察者可能因此新值而更新。
  ```
